DIFFÉRENCE ENTRE UN MCP-SERVER ET UNE API REST
Le serveur MCP permet de découvrir dynamiquement des outils par l'hote (et que l'AI va être capable d'utiliser). Dans le cas de l'API,
 il faut connaitre et configurer les routes à appeler en se basant sur une documentation.
 Concrèetement si on ajoute de nouveaux outils depuis le HOST du serveur, ceci sera automatiquement decouvert du côté de
  l'hote sans avoir à y rajouter une documentation.
--> Transport Type: STDIO

On peut avoir un server MCP depuis une image-docker, depuis un remote HTTP, ou meme en local (via la commande stdio) etc.

Un server-mcp est une sorte d'intermédiaire entre mon IA et des sources de données 

CRÉATION D'UN SERVEUR MCP pour récupérer la météo via une API (avec Open météo):
Le model de copilot est limité à 128 outils, d'où la bonne pratique de décocher les server-mcp non utiles pour tel 
ou tel fonctionnalité.
